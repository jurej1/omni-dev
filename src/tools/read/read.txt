Read the contents of a file from the local filesystem.

Usage:
- The filePath parameter must be an absolute path, not a relative path
- By default reads up to 2000 lines from the start of the file
- Use offset and limit to read specific sections of large files
- Avoid tiny repeated slices (e.g., 30-line chunks). Read a larger window if you need more context.
- Use the grep tool to find specific content in large files before reading
- If you are unsure of the correct path, use the glob tool to find it first
- You can call multiple read tools in a single response — always read files in parallel when you need several
- Contents are returned with line numbers prefixed as `<line>: <content>`
