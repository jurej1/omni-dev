A powerful content search tool built on ripgrep.

Usage:
- ALWAYS use this tool for searching file contents. NEVER invoke `grep` or `rg` via the bash tool.
- Supports full regex syntax (e.g., "log.*Error", "function\s+\w+")
- Filter files with the include parameter (e.g., "*.js", "*.{ts,tsx}")
- Returns matching file paths and line numbers sorted by modification time
- Returns up to 100 matches; if truncated, refine your pattern or path
- Pattern syntax uses ripgrep — literal braces need escaping (use `interface\{\}` to find `interface{}` in Go)
- You can call this tool in parallel with other searches in a single response
- When an open-ended search requires multiple rounds of exploration, use the Task tool instead
